GIS¶

Modules and classes:

gis.wiggly_qgis Adapter which allows using QGIS scripts (including Processing scripts) as analysis models in Wiggly.

Adapter which allows using QGIS scripts (including Processing scripts) as analysis models in Wiggly.

Requirements:
QGIS 2.4 or above A working Wiggly installation in the QGIS-bundled Python
Usage:
See the example in the examples directory.
Notes:

Don’t name your Processing scripts ‘qgis.py’ or ‘test.py’ or anything too common as those kinds of names may interfere with QGIS and cause Wiggly not to work as expected.

Your processing scripts must contain a function which has the operations you wish to analyze. In order for your processing script to be runnable by processing as well as this library, it is recommended to place all script code in a function called go() which takes in as parameters any required inputs. See the qgis example for more details

gis.wiggly_qgis.getProcessingUserScriptAsModule(scriptname)¶

Returns a reference to the module of the given Processing user script. A user script is one located in the Processing toolbox under Scripts -> User scripts.

  • scriptname: A string indicating the filename (no path or extension) of the script. E.g. ‘myscript’
gis.wiggly_qgis.wigglyCtrlInit()¶

Initializes QGIS libraries for the subprocesses created by Wiggly.

Since each subprocess is it’s own process, this function needs to be called from the file containing the analysis model. Furthermore, the function call must be protected by if __name__==’__parents_main__’.

See the qgis example.

gis.wiggly_qgis.wigglyInit(initScriptFunc)¶

Initializing routine before model execution.

  • initScriptFunc: the function handle containing the control script which sets up the UQ/SA run.
gis.wiggly_qgis.wigglyModelEnd()¶

Call this function on immmediately before the return statement of the analysis model’s

run(args=None,jobinfo=None)

function.

gis.wiggly_qgis.wigglyModelStart()¶

Call this function on the first line of the analysis model’s

run(args=None,jobinfo=None)

function. If any exceptions occurred during the

exec(wqgs.wigglyCtrlInit(),globals())

in the control script’s file, they will be thrown here.
